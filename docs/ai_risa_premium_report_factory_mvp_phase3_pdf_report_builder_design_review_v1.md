## Content Assembly Model Review

The priority order (queue fields ‚Üí existing prediction surfaces ‚Üí empty-state) is correct and safe:

- **Queue record fields** are the primary source ‚Äî these are always available for a saved row
- **Existing prediction/report surfaces** are optional ‚Äî "only if already computed and available; no new model calls, no web fetch" is a critical safety constraint
- **Empty-state placeholders** are the fallback ‚Äî no silent omission

**Quality enforcement checklist:**
- No raw JSON or internal field names in report body: defined ‚úì
- Plain-English confidence band: defined ‚úì
- Limitations statement in Section 13: defined ‚úì
- Branding/version/timestamp on every page: defined ‚úì

**Implementation note:** The phrase "existing prediction/report surfaces ‚Äî only if already computed and available" must be operationalized in the implementation slice. The implementation must define which specific fields or data structures qualify as "available prediction surfaces" and must not introduce a lazy-load or on-demand prediction call. If no prediction surface is available for a given section, the empty-state path must activate without error.

**Content assembly model verdict: PASS with one implementation-note**

## Risks and Guardrails for Future Implementation

| Risk | Guardrail |
|------|-----------|
| `prf_report_content.py` imports a module that triggers result lookup indirectly | Enforce import-level code review gate before implementation commit |
