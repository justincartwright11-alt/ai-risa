# Maintenance Trigger Threshold Note

**Effective Date:** April 18, 2026

## Maintenance/Hotfix Trigger Policy
- Any deviation from the documented contract, manifest, or go/no-go note triggers a maintenance/hotfix or rollback decision.
- No ad hoc fixes or undocumented adjustments are permitted.
- The following events require immediate classification and action:
  - Semantic mismatch in any operator outcome
  - Operator confusion or workaround behavior
  - Watchlist escalation
  - Rollback trigger
- All triggers and actions must be logged in docs/live_release_window_log.md.

## Thresholds
- Zero tolerance for contract deviation in controlled routine use.
- Any threshold breach pauses routine operations until resolved via maintenance/hotfix or rollback.
