## Changes Applied

### 1. `KNOWN_CLOSURE_STATES` constant
Added a `frozenset` of the three valid closure states:
- `effectively_closed`
- `structurally_closed`
- `theoretically_open`

Any value outside this set causes a `ValueError` at validate time and again inside `classify_finality` as defense-in-depth.

### 2. `KNOWN_FINALITY_STATES` constant
Added a `frozenset` of the three valid finality states:
- `governance_final_no_release`
- `structurally_closed`
- `temporarily_closed`

No release-enabling state (`governance_final_release`) is present or reachable.

### 3. `REQUIRED_CLOSURE_RECORD_FIELDS` constant
Defined a tuple of eight required fields that every upstream closure record must carry:
- `proposal_id`, `release_closure_assessment_id`, `closure_state`, `closure_classification`
- `release_blocker_refs`, `release_prohibition_refs`, `structural_closure_drivers`, `structurally_blocked_dependencies`

### 4. `validate_closure_record(record, index)` — new function
Called once per closure record in `build_records()` before any other processing.  
Raises `ValueError` (fail-closed) on:
- Any field in `REQUIRED_CLOSURE_RECORD_FIELDS` absent from the record
- Empty `proposal_id`
- Empty `release_closure_assessment_id`
- `closure_state` not in `KNOWN_CLOSURE_STATES`

### 5. `validate_records(records)` — new function
Called in `main()` after `build_records()` completes and before any file write.  
Raises `ValueError` (fail-closed) on:
- Duplicate `release_finality_decision_id` values
- Duplicate `proposal_id` values
- `len(records) != EXPECTED_RECORD_COUNT`
- Any `release_finality_state` not in `KNOWN_FINALITY_STATES`
- Duplicate entries in `required_remaining_resolutions`, `release_blocker_refs`, or `release_prohibition_refs` per record

### 6. Defensive guard in `classify_finality()`
Added an explicit check against `KNOWN_CLOSURE_STATES` at the top of `classify_finality()`.  
This provides defense-in-depth: the validation in `validate_closure_record` is the primary guard; this guard ensures the decision logic itself never silently defaults to `temporarily_closed` on an unknown closure state.
